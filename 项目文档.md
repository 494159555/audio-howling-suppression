## 项目概述

### 项目背景
音频啸叫抑制系统是一个基于深度学习和传统信号处理方法的音频增强项目，旨在消除音频系统中的反馈啸叫噪声，提高音频质量。项目包含多种深度学习模型（U-Net、CNN、RNN）和传统方法（移频移向、增益抑制、自适应反馈），并提供了完整的训练、评估和推理流程。

### 主要功能
- **深度学习模型**：支持多种神经网络架构进行端到端音频降噪
- **传统方法**：提供经典信号处理算法进行对比和补充
- **完整训练流程**：包含数据加载、模型训练、验证和实验管理
- **推理系统**：支持批量音频处理和实时推理
- **评估系统**：提供多维度性能指标和可视化分析

### 1. 配置模块 (config.py)

**功能**：全局配置管理，集中管理所有超参数和路径设置。

**主要配置**：

- **训练参数**：批大小(8)、学习率(1e-4)、训练轮数(50)、数据加载线程数(2)
- **调度器参数**：学习率衰减因子(0.5)、衰减等待轮数(3)

**使用方式**：
```python
from src.config import cfg

# 访问配置参数
print(cfg.SAMPLE_RATE)      # 16000
print(cfg.BATCH_SIZE)       # 8
print(cfg.DEVICE)           # cuda 或 cpu
```

### 2. 数据集模块 (dataset.py)

**功能**：音频数据集加载和预处理，支持成对的干净音频和含啸叫音频。

**核心类**：`HowlingDataset`

**主要功能**：
1. **音频加载**：使用torchaudio加载.wav文件
2. **长度归一化**：填充或截断到固定长度
3. **频谱变换**：使用STFT将时域信号转换为频谱
4. **Log变换**：对数域转换以提高数值稳定性
5. **归一化**：将值归一化到[0,1]范围
6. **维度调整**：裁剪频率维度以匹配U-Net输入

**数据流程**：
```
原始音频 [1, 48000] 
  → 长度归一化 [1, 48000]
  → STFT变换 [1, 257, 376] 
  → Log变换 [1, 257, 376]
  → 归一化 [1, 257, 376]
  → 裁剪 [1, 256, 376]
```

**使用示例**：
```python
from src.dataset import HowlingDataset
from torch.utils.data import DataLoader

# 创建数据集
dataset = HowlingDataset(
    clean_dir='data/train/clean',
    howling_dir='data/train/howling',
    sample_rate=16000,
    chunk_len=3.0
)

# 创建数据加载器
loader = DataLoader(
    dataset,
    batch_size=8,
    shuffle=True,
    num_workers=2
)

# 获取数据
noisy_mag, clean_mag = dataset[0]  # [1, 256, T], [1, 256, T]
```

### 3. 模型模块 (models/)

#### 3.1 AudioUNet5 (unet_v2.py)

**功能**：5层U-Net深度学习模型，用于音频啸叫抑制。

**网络架构**：
```
编码器 (下采样)：
  enc1: [B,1,256,T] → [B,16,128,T]
  enc2: [B,16,128,T] → [B,32,64,T]
  enc3: [B,32,64,T] → [B,64,32,T]
  enc4: [B,64,32,T] → [B,128,16,T]
  enc5: [B,128,16,T] → [B,256,8,T] (bottleneck)

解码器 (上采样) + 跳跃连接：
  dec5: [B,256,8,T] + enc4 → [B,128,16,T]
  dec4: [B,256,16,T] + enc3 → [B,64,32,T]
  dec3: [B,128,32,T] + enc2 → [B,32,64,T]
  dec2: [B,64,64,T] + enc1 → [B,16,128,T]
  dec1: [B,32,128,T] + enc1 → [B,1,256,T]
```

**特点**：
- 深度架构，强大的特征提取能力
- 跳跃连接保留多尺度信息
- Log域处理提高数值稳定性
- 掩膜机制：输出[0,1]范围的掩膜与输入相乘

**使用示例**：
```python
from src.models.unet_v2 import AudioUNet5
import torch

# 创建模型
model = AudioUNet5()

# 前向传播
input_spec = torch.randn(1, 1, 256, 376)  # [B, C, F, T]
output_spec = model(input_spec)
```

#### 3.2 AudioUNet3 (unet_v1.py)

**功能**：3层U-Net模型，参数量较少，适合快速实验。

**网络架构**：
```
编码器：
  enc1: [B,1,256,T] → [B,16,128,T]
  enc2: [B,16,128,T] → [B,32,64,T]
  enc3: [B,32,64,T] → [B,64,32,T] (bottleneck)

解码器：
  dec3: + enc2 → [B,32,64,T]
  dec2: + enc1 → [B,16,128,T]
  dec1: + enc1 → [B,1,256,T]
```

**特点**：
- 浅层网络，参数量少
- 训练速度快
- 适合原型开发和快速验证

#### 3.3 AudioCNN (CNN.py)

**功能**：纯卷积神经网络，无跳跃连接。

**网络架构**：
```
编码器：
  conv1: [B,1,256,T] → [B,32,128,T]
  conv2: [B,32,128,T] → [B,64,64,T]
  conv3: [B,64,64,T] → [B,128,32,T]
  conv4: [B,128,32,T] → [B,256,16,T]

解码器：
  upconv4: [B,256,16,T] → [B,128,32,T]
  upconv3: [B,128,32,T] → [B,64,64,T]
  upconv2: [B,64,64,T] → [B,32,128,T]
  upconv1: [B,32,128,T] → [B,1,256,T]
```

**特点**：
- 无跳跃连接，纯编码器-解码器结构
- 适合捕捉局部空间特征
- 计算复杂度较低

#### 3.4 AudioRNN (RNN.py)

**功能**：结合CNN和LSTM的混合模型，处理时序依赖关系。

**网络架构**：
```
前置CNN：
  [B,1,256,T] → [B,64,256,T]

双向LSTM：
  [B*256,T,64] → [B*256,T,256] (hidden_size*2)

后置CNN：
  [B,256,256,T] → [B,64,256,T] → [B,32,256,T] → [B,1,256,T]
```

**特点**：
- LSTM处理时序信息，捕捉长期依赖
- 双向LSTM同时考虑过去和未来
- 每个频率bin独立进行时序处理
- 适合处理具有时间相关性的音频信号

### 4. 传统方法模块 (traditional/)

#### 4.1 移频移向法 (frequency_shift.py)

**原理**：通过轻微的频率偏移破坏反馈系统的相位条件，从而抑制啸叫。

**参数**：
- `shift_hz`: 频率偏移量，默认20Hz
- `sample_rate`: 采样率，默认16000Hz

**特点**：
- 计算简单，实时性好
- 音质影响小
- 适合轻微啸叫场景

**使用示例**：
```python
from src.traditional import FrequencyShiftMethod

method = FrequencyShiftMethod(shift_hz=20.0)
output = method(noisy_mag)  # [B, 1, F, T]
```

#### 4.2 增益抑制法 (gain_suppression.py)

**原理**：检测啸叫频段并应用增益抑制。

**参数**：
- `threshold_db`: 抑制阈值，默认-30dB
- `reduction_factor`: 抑制因子，默认0.1

**特点**：
- 针对性强，效果明显
- 可能影响有用信号
- 适合明显啸叫场景

**使用示例**：
```python
from src.traditional import GainSuppressionMethod

method = GainSuppressionMethod(threshold_db=-30.0)
output = method(noisy_mag)
```

#### 4.3 自适应反馈抵消法 (adaptive_feedback.py)

**原理**：使用自适应滤波器估计反馈路径并消除反馈。

**参数**：
- `filter_length`: 滤波器长度，默认64
- `step_size`: 步长，默认0.01
- `leakage`: 泄漏因子，默认0.99

**特点**：
- 自适应能力强
- 效果稳定
- 计算复杂度较高
- 适合复杂环境

**使用示例**：
```python
from src.traditional import AdaptiveFeedbackMethod

method = AdaptiveFeedbackMethod(filter_length=64)
output = method(noisy_mag)
```

### 7. 推理模块 (inference.py)

**功能**：使用训练好的模型对音频进行啸叫抑制推理。

**支持的相位重建方法**：
1. **Griffin-Lim算法**（默认）：更好的音质，但速度较慢
2. **ISTFT算法**：速度快，使用原始相位

**使用方法**：

#### 命令行使用
```bash
# 使用Griffin-Lim（推荐）
python inference.py \
  --model experiments/exp_xxx/checkpoints/best_model.pth \
  --input input.wav \
  --output output.wav \
  --device auto \
  --use_griffin_lim

# 使用ISTFT（更快）
python inference.py \
  --model experiments/exp_xxx/checkpoints/best_model.pth \
  --input input.wav \
  --output output.wav \
  --device auto
```

#### 代码调用
```python
from src.models import AudioUNet5
import torch

# 加载模型
model = AudioUNet5()
model.load_state_dict(torch.load('model.pth'))
model.eval()

# 推理
output = model(input_tensor)
```

**处理流程**：
1. 加载模型和音频
2. 重采样到16kHz（如需要）
3. STFT频谱分析
4. 预处理（Log变换、归一化、裁剪）
5. 模型推理
6. 后处理（反归一化、反Log变换）
7. 相位重建（Griffin-Lim或ISTFT）
8. 智能归一化（防止削波）
9. 保存输出

### 8. 评估系统 (evaluation/)

#### 8.1 指标计算 (metrics.py)

**功能**：计算多种音频质量指标。

**支持的指标**：
- **SNR改善 (dB)**：信噪比改善程度
- **PSNR (dB)**：峰值信噪比
- **STOI (0-1)**：短时客观可懂度
- **MOS分数 (1-5)**：主观音质评分估算
- **啸叫衰减率 (dB)**：高频段能量衰减
- **频谱平滑度改善**：频谱连续性改善

**使用示例**：
```python
from src.evaluation.metrics import AudioMetrics, calculate_mos_score

metrics_calculator = AudioMetrics()

# 计算指标
snr_improvement = metrics_calculator.calculate_snr(clean, enhanced, noisy)
psnr = metrics_calculator.calculate_psnr(clean, enhanced)
stoi = metrics_calculator.calculate_stoi(clean, enhanced)

# 计算MOS分数
metrics = {
    'snr_improvement_db': snr_improvement,
    'psnr_db': psnr,
    'stoi_score': stoi,
    'howling_reduction_db': 5.0
}
mos_score = calculate_mos_score(metrics)
```

## 配置参数说明

### 音频处理参数

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| `SAMPLE_RATE` | 16000 | 音频采样率(Hz) |
| `CHUNK_LEN` | 3.0 | 音频片段长度(秒) |
| `CHUNK_SIZE` | 48000 | 音频片段长度(采样点数) |
| `N_FFT` | 512 | FFT窗口大小 |
| `HOP_LENGTH` | 128 | STFT跳跃长度 |

**参数说明**：
- **采样率**：16kHz适合语音处理，平衡质量和计算复杂度
- **音频长度**：3秒包含足够上下文信息，同时保持训练效率
- **FFT窗口**：512点提供足够的频率分辨率
- **跳跃长度**：128点（N_FFT的1/4）保证时频分辨率平衡

---

### 训练超参数

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| `BATCH_SIZE` | 8 | 批大小 |
| `LEARNING_RATE` | 1e-4 | 学习率 |
| `NUM_EPOCHS` | 50 | 训练轮数 |
| `NUM_WORKERS` | 2 | 数据加载线程数 |

**调整建议**：
- **BATCH_SIZE**：根据GPU内存调整，显存不足可减小到4或2
- **LEARNING_RATE**：默认1e-4适合Adam优化器，可根据训练情况调整
- **NUM_EPOCHS**：50轮通常足够，可监控验证集Loss提前停止
- **NUM_WORKERS**：根据CPU核心数调整，通常2-4个即可

---

### 学习率调度参数

| 参数名 | 默认值 | 说明 |
|--------|--------|------|
| `SCHEDULER_FACTOR` | 0.5 | 学习率衰减因子 |
| `SCHEDULER_PATIENCE` | 3 | 学习率衰减等待轮数 |

**调度策略**：ReduceLROnPlateau
- 当验证集Loss在3轮内无改善时，学习率减半
- 最小学习率：1e-6

## 常见问题

### Q1: 训练时显存不足怎么办？

**解决方案**：
1. 减小`BATCH_SIZE`：从8减到4或2
2. 使用更小的模型：AudioUNet3替代AudioUNet5
3. 使用CPU训练：设置`CUDA_VISIBLE_DEVICES=""`

### Q2: 训练Loss不下降怎么办？

**可能原因**：
1. 学习率过大或过小
2. 数据集问题
3. 模型架构问题

**解决方案**：
1. 检查学习率，尝试调整为1e-5或1e-3
2. 检查数据集，确保clean和howling文件对应
3. 使用TensorBoard监控Loss曲线
4. 尝试不同的模型架构

### Q3: 推理结果音质不好怎么办？

**可能原因**：
1. 模型训练不充分
2. 相位重建方法选择不当
3. 输入音频参数不匹配

**解决方案**：
1. 使用Griffin-Lim算法（`--use_griffin_lim`）
2. 检查输入音频采样率是否为16kHz
3. 重新训练模型，增加训练轮数
4. 尝试不同的模型架构

### Q4: 如何选择最佳模型？

**建议流程**：
1. 训练多个模型（不同架构、不同超参数）
2. 使用`src/evaluate.py`在验证集上评估
3. 对比Log-L1 Loss和其他指标
4. 选择验证集Loss最低的模型

### Q5: 如何处理不同长度的音频？

**解决方案**：
1. **训练时**：使用`HowlingDataset`自动填充或截断
2. **推理时**：
   - 长音频：分段处理（每段3秒），重叠拼接
   - 短音频：自动填充到3秒
3. **代码示例**：

### Q7: 如何加速训练？

**优化策略**：
1. **使用GPU**：确保CUDA可用
2. **增大BATCH_SIZE**：在显存允许的情况下
3. **增加NUM_WORKERS**：根据CPU核心数调整
4. **使用混合精度训练**：
```python
from torch.cuda.amp import autocast, GradScaler

scaler = GradScaler()

for noisy_mag, clean_mag in train_loader:
    optimizer.zero_grad()
    
    with autocast():
        pred_mag = model(noisy_mag)
        loss = criterion(pred_mag, clean_mag)
    
    scaler.scale(loss).backward()
    scaler.step(optimizer)
    scaler.update()
```
