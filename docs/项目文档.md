## 项目概述

音频啸叫抑制系统是一个基于深度学习的音频增强项目，用于消除音频系统中的反馈啸叫噪声。项目支持13+种U-Net变体架构、多种训练策略和评估方法。

### 主要功能
- **深度学习模型**：13+种U-Net变体及CNN、RNN基线模型
- **训练策略**：混合精度、课程学习、多种学习率调度器
- **损失函数**：L1、MSE、频谱损失、多任务损失、对抗损失
- **数据增强**：音频增强、SpecAugment、Mixup、对抗增强
- **后处理**：自适应阈值、多帧平滑、增益控制
- **传统方法**：移频移向、增益抑制、自适应反馈
- **评估系统**：多维度指标、可视化分析、基准测试

## 核心模块

### 1. 配置模块 (config.py)

全局配置管理，包含所有默认参数、模型注册和路径配置。

**主要配置**：
- 路径配置：项目根目录、数据路径、实验输出路径
- 音频参数：采样率(16kHz)、片段长度(3秒)、FFT窗口(512)、跳跃长度(128)
- 训练参数：批大小(8)、学习率(1e-4)、轮数(50)
- 模型注册：13种U-Net变体映射及描述
- 损失函数配置：6种损失函数类型
- 训练策略：混合精度、课程学习、学习率调度器
- 后处理配置：自适应阈值、平滑、增益控制

```python
from src.config import cfg

# 使用配置
print(cfg.SAMPLE_RATE)      # 16000
print(cfg.DEVICE)           # cuda 或 cpu
print(cfg.AVAILABLE_MODELS)  # 模型字典
```

### 2. 数据集模块 (dataset.py)

**HowlingDataset** 类实现音频数据加载和预处理。

**数据流程**：
```
原始音频 → 长度归一化 → STFT → Log变换 → 归一化 → 裁剪
[1, 48000] → [1, 48000] → [1, 257, 376] → [1, 257, 376] → [1, 257, 376] → [1, 256, 376]
```

**功能**：
- 自动加载成对音频（clean/howling）
- 固定长度处理（填充/截断）
- STFT频谱变换
- Log域转换和归一化
- 支持数据增强

### 3. 损失函数模块 (models/loss_functions.py)

提供多种损失函数：

| 损失函数 | 说明 |
|---------|------|
| **L1Loss** | L1距离损失 |
| **MSELoss** | 均方误差损失 |
| **SpectralLoss** | 对数域频谱损失 |
| **SpectralConsistencyLoss** | 频谱一致性损失（平滑性约束） |
| **MultiTaskLoss** | 多任务组合损失 |
| **AdversarialLoss** | GAN对抗损失（含判别器） |

**使用示例**：
```python
from src.models.loss_functions import MultiTaskLoss

criterion = MultiTaskLoss(
    weights={'spectral': 0.5, 'l1': 0.3, 'mse': 0.2},
    use_consistency=True
)
total_loss, loss_dict = criterion(pred, target)
```

### 4. 数据增强模块 (models/augmentation.py)

提供多种数据增强策略：

| 增强方法 | 说明 |
|---------|------|
| **AudioAugmentation** | 音频级增强（噪声、音调、位移） |
| **SpecAugment** | 频谱图掩蔽（频率/时间掩蔽） |
| **MixupAugmentation** | 样本混合增强 |
| **AdversarialAugmentation** | 对抗性增强 |
| **CombinedAugmentation** | 组合增强策略 |

### 5. 训练策略模块 (models/training_strategies.py)

高级训练策略：

| 策略 | 说明 |
|-----|------|
| **MixedPrecisionTrainer** | 混合精度训练（AMP） |
| **CosineAnnealingWarmupScheduler** | 余弦退火+预热调度器 |
| **OneCycleScheduler** | 单周期学习率策略 |
| **CurriculumLearning** | 课程学习（渐进难度） |

**学习率调度器类型**：
- `plateau`: ReduceLROnPlateau
- `cosine_warmup`: 余弦退火+预热
- `one_cycle`: 单周期策略
- `step`: 步进衰减

### 6. 后处理模块 (models/post_processing.py)

推理后处理优化：

| 后处理方法 | 说明 |
|-----------|------|
| **AdaptivePostProcessing** | 自适应阈值处理 |
| **MultiFrameSmoother** | 多帧平滑（移动平均/Kalman/维纳/中值） |
| **AdaptiveGainControl** | 自适应增益控制（AGC/DRC/限幅器） |
| **PostProcessingPipeline** | 组合后处理流水线 |

## 模型架构

### U-Net变体 (13种)

| 版本 | 名称 | 特点 |
|-----|------|------|
| **v1** | AudioUNet3 | 3层基线U-Net |
| **v2** | AudioUNet5 | 5层基线U-Net（默认） |
| **v3** | AudioUNet5Attention | + 注意力门 |
| **v4** | AudioUNet5Residual | + 残差连接 |
| **v5** | AudioUNet5Dilated | + 空洞卷积 |
| **v6** | AudioUNet5Optimized | 注意力+残差+空洞（组合优化） |
| **v7** | AudioUNet5LSTM | + 双向LSTM时序建模 |
| **v8** | AudioUNet5TemporalAttention | + 时间注意力 |
| **v9** | AudioUNet5ConvLSTM | + ConvLSTM层 |
| **v10** | AudioUNet5GAN | + GAN框架（判别器） |
| **v11** | AudioUNet5MultiScale | 多尺度特征提取 |
| **v12** | AudioUNet5Pyramid | 金字塔池化 |
| **v13** | AudioUNet5FPN | 特征金字塔网络 |

### 基线模型

- **AudioCNN**: 4层卷积自编码器（无跳跃连接）
- **AudioRNN**: CNN + 双向LSTM混合架构

### 模型选择建议

| 场景 | 推荐模型 |
|-----|---------|
| 快速原型 | unet_v1 |
| 标准使用 | unet_v2 |
| 追求性能 | unet_v6_optimized |
| 时序问题 | unet_v7_lstm / unet_v8_temporal_attention |
| 复杂场景 | unet_v10_gan |
| 多尺度特征 | unet_v11_multiscale / unet_v13_fpn |

## 训练与评估

### 训练命令

```bash
# 基础训练（使用默认模型unet_v2）
python src/train.py

# 指定模型
python src/train.py --model unet_v6_optimized

# 使用YAML配置文件（推荐）
python src/train.py --config configs/unet_v3_attention.yaml

# 覆盖配置参数
python src/train.py --config configs/unet_v2.yaml --lr 2e-4 --batch-size 4 --epochs 100

# 自定义实验名称
python src/train.py --model unet_v6_optimized --exp-name "my_experiment"
```

### 配置优先级

```
CLI参数 > YAML配置 > config.py默认值
```

### 监控训练

```bash
# 启动TensorBoard
tensorboard --logdir experiments/

# 访问 http://localhost:6006
```

### 运行测试

```bash
# 运行所有测试
python tests/run_tests.py
```

### 评估模型

```bash
# 评估训练好的模型
python src/evaluate.py --checkpoint experiments/exp_xxx/checkpoints/best_model.pth

# 快速评估传统方法
python -c "from src.evaluation.test_runner import run_quick_evaluation; run_quick_evaluation(methods=['frequency_shift', 'gain_suppression'], num_samples=5)"
```

### 推理

```bash
# 单文件推理（Griffin-Lim，更好音质）

# 快速推理（ISTFT）
python inference.py --model experiments/exp_xxx/checkpoints/best_model.pth --input input.wav --output output.wav --device auto
```

## 配置系统

### 配置文件结构

项目采用三层配置系统：

1. **全局配置** (`src/config.py`): 默认参数和模型注册
2. **YAML配置** (`configs/*.yaml`): 模型特定配置
3. **CLI参数**: 命令行覆盖

### YAML配置示例

```yaml
# configs/unet_v3_attention.yaml
model:
  name: unet_v3_attention
  class: AudioUNet5Attention

training:
  batch_size: 8
  learning_rate: 1e-4
  epochs: 50
  loss_function: multitask

loss:
  type: multitask
  weights:
    spectral: 0.5
    l1: 0.3
    mse: 0.2

training_strategies:
  mixed_precision: true
  lr_scheduler: cosine_warmup
  warmup_epochs: 5

data_augmentation:
  enabled: true
  spec_augment:
    freq_masking: true
    time_masking: true
```

### 模型注册

在 `src/config.py` 中注册新模型：

```python
AVAILABLE_MODELS = {
    'unet_vXX_name': 'ModelClassName',
    # ...
}

MODEL_DESCRIPTIONS = {
    'unet_vXX_name': '模型描述',
    # ...
}
```

## 常见问题

### Q: CUDA显存不足？
**解决**：减小 `batch_size` (8→4→2) 或使用更小模型 (unet_v1)

### Q: 训练Loss不下降？
**检查**：学习率、数据配对、TensorBoard可视化

### Q: 推理音质不好？
**尝试**：使用 `--use_griffin_lim`、检查采样率(16kHz)、增加训练轮数

### Q: 如何选择最佳模型？
**流程**：训练多个模型 → 评估验证集 → 对比Loss → 选择最佳

### Q: CPU训练？
**设置**：
```bash
export CUDA_VISIBLE_DEVICES=""
# 或在代码中 device = torch.device("cpu")
```
